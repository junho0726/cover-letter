자기 소개

유지보수 효율성과 안정적인 서비스 운영을 최우선으로 고려하는 4년 차 백엔드 엔지니어입니다.

약 4년 동안 13개의 프로젝트를 수행하며 금융, 이커머스, 공공기관 시스템 등 다양한 도메인의 실서비스를 설계하고 운영해왔습니다. 수동 배포 중심의 SFTP 환경을 Jenkins 기반 CI/CD 파이프라인으로 전환하여 배포 안정성과 효율을 개선했고, AWS EC2와 Nginx를 활용한 서버 구축 및 HTTPS 설정을 직접 담당하며 서비스 운영 경험을 쌓았습니다.

또한 .NET 기반 레거시 API를 Java 환경으로 재설계하며 성능과 유지보수성을 개선하는 등, 기존 시스템을 더 안정적이고 관리하기 쉬운 구조로 전환하는 데 강점을 가지고 있습니다. 복잡해진 시스템 구조를 정리하고 배포·운영 부담을 줄이는 방향으로 개선해왔으며, 서비스 확장에도 안정적으로 대응할 수 있는 구조를 만드는 개발자를 지향합니다.

핵심 역량

  • 백엔드 API 설계 및 개발
    Spring 기반 API 서버 개발 경험과 다양한 도메인의 실서비스 구현 경험 보유
  • 배포 자동화 및 운영 환경 구축
    SFTP 기반 수동 배포를 Jenkins CI/CD 환경으로 전환하고, AWS EC2 및 Nginx를 활용한 서버 구축 및 HTTPS 설정 경험
  • 레거시 시스템 개선
    .NET 기반 API를 Java 환경으로 재설계하며 성능 및 유지보수성 개선
  • 문제 해결 및 구조 개선 능력
    운영 환경 제약 속에서도 시스템 구조를 재설계하여 안정적으로 서비스 오픈 경험

문제 해결 경험

배포 직전, 운영 인프라와 개발 환경 간의 불일치를 확인한 경험이 있습니다. Java 8과 Spring Boot 기반으로 시스템 개발을 완료했지만, 실제 운영 서버는 Java 7 기반의 JEUS 환경이었고 전자정부프레임워크 표준을 준수해야 하는 제약이 있었습니다.

초기 단계에서 운영 환경을 충분히 검증하지 못한 것이 원인이었고, 이로 인해 java.time 패키지 사용 불가, Spring Boot Auto Configuration 사용 불가 등의 문제가 발생했습니다. 결과적으로 기존 구조 그대로는 배포가 어려운 상황이었고, 오픈 일정이 지연될 수 있는 위기에 직면했습니다.

긴급 대응 과정
  • LocalDateTime 기반 날짜 로직을 Java 7 환경에 맞게 Date, Calendar 구조로 전면 수정
  • 반복되는 날짜 처리 로직을 공통 유틸로 분리하여 안정성 확보
  • Spring Boot 기반 의존성 제거 후 전자정부프레임워크 구조에 맞게 XML 기반 Bean 및 DataSource 설정으로 재구성
  • Jackson 등 라이브러리 충돌 문제를 해결하고, 호환 가능한 버전으로 의존성 재정리
  • 재발 방지를 위해 개발 착수 전 운영 서버 환경(JDK, WAS 등) 사전 확인 체크리스트 제안 및 적용

주말을 포함한 약 2일간의 집중 대응을 통해 기술적 문제 없이 일정 내 서비스 오픈을 완료했습니다. 이 경험을 통해 인프라 사전 검증이 프로젝트 성공에 얼마나 큰 영향을 주는지 체감했고, 제약이 큰 환경에서도 비즈니스 연속성을 지켜내는 실무 대응력의 중요성을 다시 확인할 수 있었습니다.

입사 후 포부

지난 3년 10개월 동안 금융권 시스템 운영 개선부터 AI 기반 RAG 서비스 프로토타입 개발까지 다양한 프로젝트를 경험하며 백엔드 개발 역량을 쌓아왔습니다. 단순한 기능 유지보수에 머무르지 않고, 복잡한 권한 체계를 정리하거나 레거시 API를 Java 기반 구조로 재설계하는 등 시스템을 더 안정적이고 관리하기 쉬운 형태로 개선하는 데 집중해왔습니다.

운영 환경에서 발생하는 데이터 불일치와 배포 이슈를 직접 해결하며, 초기 설계의 완성도뿐 아니라 운영 과정에서의 효율성과 관리 용이성 또한 중요하다는 점을 체감했습니다. 이러한 실무 경험을 통해 문제를 단순히 해결하는 것을 넘어, 구조적으로 개선하는 방향으로 접근하게 되었고 이는 저에게 큰 자산이 되었습니다.

입사 후에는 이러한 경험을 바탕으로 확장성과 유지보수성을 함께 고려한 백엔드 구조를 설계하고 싶습니다. 트래픽 증가나 기능 확장 상황에서도 안정적으로 대응할 수 있도록 DB 구조와 서버 로직을 지속적으로 점검하고 개선하겠습니다.

또한 팀과의 협업 과정에서 구조에 대한 고민과 개선 사항을 공유하고 기록하며, 장기적으로 안정적인 서비스를 만들어가는 개발자로 성장하겠습니다.
